1. mac+hexo+github搭建博客

Posted on 2016-03-27

一直以来，自己不断更换博客。用过的博客有网易，新浪，csdn，博客园，甚至还用过qq空间，还用过有道云笔记，微软onenote等等。每次迁移都是因为对自己的博客不满意，感觉自己的博客质量太差。有的博客内容仅仅是学习重点摘要，有的博客文章全篇都是摘录别人的，有的文章全是代码.

–

因此给自己定下规矩：

1.在这个博客中，禁止大量摘录别人的东西，甚至全篇抄袭，可以引用别人的网址。

2.必须所有的东西都必须原创，将那些杂七杂八的文章都写在其他博客上。

3.每次想好开一个系列，必须把这个系列写完。

4.文章必须自己看着舒服，别人看着也舒服，自己若干年后再看自己的博客还会很开心。

5.每周必须写一篇博客。

6.这次博客必须系统化,不能东一榔头，西一榔头地写。这点非常重要，决定了自己会不会喜欢自己的博客。

7.这个博客的内容不在于多，而在于精。

8.博客的技术最好都引用官方博客，即使看别人写的也要注意看他从哪里引来的。

9.记录自己的学习过程。

–

配置安装开始之前：

安装hexo以及配置github花了2天的时间，现在我把我按照官方的方法走的过程记录下。仅作参考，因为随着hexo版本的更新，一些命令之类的可能会变。

安装任何环境及软件的基本方法:

第一反应按照官方的的步骤走。官方，官方，官方，重要的事说三遍。有的需要自备梯子，我用的vpn是免费的GreenVPN(官网)。

第二若果发现还是下载不下来的话，就看看有没有国内的镜像。国内网站记得看版本是不是最新的。

第三终端权限不够加sudo。

第三如果这条路还走不下去，就试着重新换种方法，环境，或者放弃重新选条路。

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。

注意：
下面的< folder > 自己写，我这里写的是 “neededfile”.
init 是新建网站。

新建完成后，”neededfile”的目录如下：

config.yml
package.json
scaffolds
source
themes

–

source资源文件夹是存放用户资源的地方。 里面的_posts 文件夹是存放自己写的markdown的md文件。Markdown 和 HTML 文件会被hexo解析并放到 public 文件夹，而其他文件会被拷贝进public文件中的.每次解析后并添加的public文件夹里的css,js ,index.html就是最重要的东西。我每次都把他拷进github同步的文件夹名为quzhanlei.github.io的文件夹中。

–

3.新建文章

新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来

  • 新建文章，生成的md文件就在source文件下的_posts的文件夹里。然后就可以通过markdown软件进行操作。
  • 生成静态文件，就是生成public文件夹的东西
  • 发布草稿（一般不用）
  • 启动本地服务器。默认情况下，访问网址为：http://localhost:4000/。

每次新建文章之前先：
清除缓存文件 (db.json) 和已生成的静态文件 (public)
